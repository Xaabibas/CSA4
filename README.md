## Язык программирования `Alg`

### Синтаксис языка
```
<program> ::= <decl> | <decl> <program>

<decl> ::= <var_decl> | <func_decl> | <ifunc_decl> | <statement>

<var_decl> ::= "var" <identifier> ";"
               | "var" <identifier> "=" <expression> ";"

<func_decl> ::= "func" <identifier> "(" [ <parametr_list> ] ")" <block>
<parametr_list> ::= <identifier> | <identifier> "," <parametr_list>
<ifunc_decl> ::= "ifunc" "(" <number> ")" <block>

<block> ::= "{" <statement_sequence> "}"
<statement_sequence> ::= <statement> | <statement> <statement_sequence>
<statement> ::= <var_decl>
                | <assignment>
                | <if_statement>
                | <while_statement>
                | <return_statement>
                | <func_call> ";"

<assignment> ::= <identifier> "=" <expression> ";"

<if_statement> ::= "if" "(" <condition> ")" <block>
                   [ "else" <block> ]

<while_statement> ::= "while" "(" <condition> ")" <block>

<return_statement> ::= "return" [ <expression> ] ";"

<func_call> ::= <identifier> "(" [ <argument_list> ] ")"
<argument_list> ::= <expression> | <expression> "," <argument_list>

<condition> ::= <expression> <bool_op> <expression>
<bool_op> ::= "=="
              | "!="
              | "<"
              | "<="
              | ">"
              | ">="

<expression> ::= <term>
               | <expression> "+" <term>
               | <expression> "-" <term>
               | <string>

<term> ::= <factor>
           | <term> "*" <factor>

<factor> ::= <number>
             | <identifier>
             | "(" <expression> ")"
             | <func_call>
           
<identifier> ::= <letter>
                 | <identifier> <letter>
                 | <identifier> <digit>
                 | <identifier> "_"

<number> ::= <digit>
             | <number> <digit>

<letter> ::= "a" | ... | "z"
             | "A" | ... | "Z"

<digit> ::= "0" | ... | "9"

<string> ::= "\"" [ <string_content> ] "\"" 

<string_content> ::= <sym> | <string_content> <sym>

<sym> ::= "letter" | "digit" | " " | ...
```

## Организация памяти

Система построена по Пристанской архитектуре, то есть нет разделения на память команд и память данных. Память работает в линейном, плоском адресном пространстве


Обращение к памяти происходит по регистру `AR`

## Система команд

### Особенности процессора

Машинное слово - 32 бита

Размер всех команд - 32 бита

Так как процессор имеет аккумуляторную архитектуру большинство инструкций взаимодействуют с регистром `AC`

Процессор имеет аппаратную поддержу стека
  - регистр `SP` указывает на вершину стека. 
  - инструкция `PUSH` кладет значение аккумулятора на стек
  - инструкция `POP` инкриминтирует значение указателя стека `SP`. При этом `POP` не изменяет значение аккумулятора
  - обращение к данным на стеке может быть организовано с помощью режима адресации `RELATIVE_SP`. Так, например, чтобы загрузить значение с вершины стека в аккумулятор можно использовать команду `LOAD &(0)`

Цикл исполнения команды:
  - Этап `instruction fetch` - цикл выборки команды. По адресу `IP` загружается значение в `CR`, значение `IP` инкриминтируется
  - Этап `operand fetch` - цикл выборки операнда. Пропускается для безадресных команд. Происходит в зависимости от режима адресации команды. После завершения цикла в регистре `DR` расположен операнд 
  - Этап `execution` - цикл выполнения. Непосредственно выполнение команды
  - Этап `interrupt` - цикл прерывания. Процессор проверяет контроллер ввода-вывода на запрос прерывания и в случае, если запрос имеется, переходит к его обработке. При обработке имеющегося прерывания или в режиме запрета прерываний этап пропускается 

### Обработка прерывания

При запросе на прерывания процессор сохраняет контекст выполнения и переходит по адресу обработчика прерываний:
  - На вершину стека кладется регистр `PS`
  - На вершину стека кладется регистр `IP`
  - Из памяти по адресу порта устройства, запросившего прерывание (`mem[port]`), читается адрес обработчика прерывания и загружается в регистр `IP`
  - Процессор переходит к обработке прерывания

В конце обработки прерывания процессором должна быть выполнена команда `IRET`, которая:
  - Снимает значение со стека (сохраненный адрес следующей команды) и помещает его в регистр `IP`
  - Снимает значение со стека (сохраненный статус программы) и помещает его в регистр `PS`

### Кодирование инструкций

Инструкция строится следующим образом:
  - биты $0, \dots, 19$ - аргумент команды `arg` (адрес, смещение или непосредственно операнд)
  - биты $20, \dots, 23$ - режим адресации `addr_mode` (тип операнда)
  - биты $24, \dots, 31$ - код команды `op_code`

Инструкции могут быть адресными (имеющими операнд) и безадресными. Если функция адресная, то она проходит этап `operand fetch`, в противном случае она сразу переходит к этапу выполнения
Безадресные команды имеют биты $28, \dots, 31$ в значении ноль. То есть формат безадресной команды - `0x0XXXXXXX`

Так как инструкции и данные хранятся в одной памяти, они могут быть конвертированы друг в друга

Режимы адресации:
  - `DIRECT` - прямая адресация (`mem[arg]`)
  - `RELATIVE_IP` - относительно `IP` (`mem[IP + arg]`)
  - `RELATIVE_SP` - относительно `SP` (`mem[SP + arg]`)
  - `RELATIVE_INDIRECT` - косвенная непрямая адресация (`mem[mem[arg]]`)
  - `DIRECT_LOAD` - прямая загрузка операнда (`arg`)

## Транслятор 

### Интерфейс командной строки

```
python Translator.py <source_file> <dest_file> <memory_file> [<debug_file>]
```
Реализовано в модуле [Translator.py](Translator.py)

Транслятор преобразует текст программы в древовидную структуру. Затем производит последовательные преобразования этой структуры. В итоге генерируется список инструкций

### Этапы трансляции
0. Подключение стандартной библиотеки [stdlib](stdlib) набором подготовленных функций
1. Предобработка - преобразование исходного текста в последовательность токенов. Реализуется модулем [Lexer.py](Lexer.py)
2. Преобразование последовательного списка токенов в древовидную структуру, начальный анализ на соответствие синтаксису. Реализуется в модуле [AST.py](AST.py)
3. Обход полученного дерева с анализом на соответствие семантическим правилам. Реализуется в модуле [Analyzer.py](Analyzer.py)
4. Сбор имен переменных, функций и обработчиков прерываний, сопоставление им адресов в памяти машины
5. Обход семантического дерева с генерацией машинного кода
    - Обход дерева начинается с функций и обработчиков прерываний
    - После обхода функций и обработчиков прерываний начинается обход основного кода

Правила генерации машинного кода:
  - При встрече числового значения генерируется команда загрузки
    ```
    num;
    
    LD #num;
    ```
    Так как операнд ограничен 20 битами, число может быть в диапазоне $[-2^19; 2^19 - 1]$. Но при этом сами переменные могут принимать значения в диапазоне $[-2^31; 2^31 - 1]$, поэтому для получения числа вне из разрешенного диапазона можно использовать арифметические операции: `1024 * 1024 * 1024`
  - При встрече символьного значения генерируется соответствующее ему числовой значение
    ```
    char;
    
    LD #ord(char);
    ```
  - При встрече строкового значения генерируется соответствующий ему адрес строки
    ```
    str;
    
    LD #string[str];
    ```
  - При встрече оператора обращения по памяти вычисляется адрес, по которому происходит чтение, сохраняемый во временную переменную. После чего косвенной непрямой адресацией значение читается
    ```
    &(expr);
    
    generate(expr)
    ST tmp_addr
    ST ((tmp_addr))
    ```
  - Бинарные операции предполагают вычисление левой и правой части, после чего происходит соответствующая операция
    ```
    expr1 + expr2;
    
    generate(expr2)
    ST tmp_addr
    generate(expr1)
    ADD tmp_addr
    ```
    Вместо оператора `+` может находиться любой бинарный оператор
  - При встрече вызова функции происходит генерация вызова функции
    ```
    function();
    
    CALL function_addr
    ```
  - При встрече оператора присваивания переменной генерируется значение, присваиваемое переменной, которое потом пишет по соответствущему переменной адресу
    ```
    x = expr;
    
    generate(expr)
    ST x_addr
    ```
    Для глобальных переменных происходит прямое присваивание, для локальных - изменение значения на стеке
  - При встрече оператора присваивания по адресу вычисляется адрес, по которому происходит присваивание, сохраняемый во временную переменную. Затем генерируется присваиваемое значение. После чего косвенной непрямой адресацией значение записывается 
    ```
    &(expr1) = expr2;
    
    generate(expr1)
    ST tmp_addr
    generate(expr2)
    ST ((tmp_addr))
    ```
  - При встрече условия генерируется соответсвующее выражение с командой `COMP`, выставляющей флаги. После этого генерируется команда ветвления, соответствующая условному оператору
    ```
    (expr1 == expr2)
    
    generate(expr1)
    ST tmp_addr
    generate(expr2)
    COMP tmp_addr
    JNE end_block
    ```
    Значение `end_block` определяется в конце генерации блока. Вместо оператора `==` может стоять любой условный оператор, из-за чего может измениться порядок загрузки выражений и команда перехода
  - При встрече условного оператора `if` генерируется условие. После этого генерируется блок кода, исполняемый при выполнении условия. В конце, обновляется значение `end_block` для команды перехода
    ```
    if (cond) {
        then_block
    }
    
    generate(cond)
    generate(then_block)
    ```
    Если условный оператор имеет блок `else` то в конце генерации `then_block` добавляется безусловный переход. Затем генерируется `else_block` 
    ```
    if (cond) {
        then_block
    } else {
        else_block
    }
    
    generate(cond)
    generate(then_block)
    JMP end_else_block
    generate(else_block)
    ```
  - При встрече цикла `while` генерируется условие. После этого генерируется тело цикла. В конце тела генерируется команда возвращения к условию. После этого обновляется значение `end_block` для команды перехода
    ```
    while (cond) {
        then_block
    }
    
    generate(cond)
    generate(block)
    JMP cond_addr
    ```
  - При встрече `return` генерируется выражение, если оно есть, и происходит выход из функции
    ```
    return expr;
    
    generate(expr)
    RET
    ```
    Также происходит сброс значение локальных переменных со стека
  - При встрече функции `func` генерируется тело функции:
    ```
    func function() {
        block
    }
    
    generate(block)
    ```
    Если функция не вызывает `return`, то он добавляется автоматически. Также происходит сброс значение локальных переменных со стека
  - При встрече обработчика `ifunc` генерируется тело функции:
    ```
    ifunc function() {
        block
    }
    
    generate(block)
    IRET
    ```
    В конце происходит в=вызов команды `IRET`. Также происходит сброс значение локальных переменных со стека
  - При встрече `@` генерируется команда `IN` по порту 0
  - При встрече `% expr` генерируется значение `expr`, после чего вызывается функция `OUT` по порту 1
  - При встрече `!$` генерируется команда `DI`
  - При встрече `$` генерируется команда `EI`

## Тестирование

Тестирование осуществляется с помощью golden test-ов, реализованных для следующих программ:
  - [hello.yml](golden/hello.yml) - напечатать `Hello, World!`
  - [cat.yml](golden/cat.yml) - напечатать данные, поданные на сход симулятору через файл ввода
  - [hello_user_name.yml](golden/hello_user_name.yml) - запросить у пользователя имя, считать его и вывести приветствие
  - [sort.yml](golden/sort.yml) - пользователь через файл ввода загружает в систему список чисел, в ответ выводит отсортированные данные
  - [64.yml](golden/64.yml) - арифметика двойной точности - демонстрация работы с 64-битными числами
  - [prob1.yml](golden/prob1.yml) - Euler problem 4 [link](https://projecteuler.net/problem=4)

Скрипт выполнения интеграционных тестов - [integration_test.py](integration_test.py)

CI сконфигурирован с помощью GitHub Actions:
```
name: CI

on:
  push:
    branches: [ main, master ]
  pull_request:
    branches: [ main, master ]

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v3

    - name: Set up Python
      uses: actions/setup-python@v4
      with:
        python-version: '3.11'

    - name: Install dependencies
      run: |
        python -m pip install --upgrade pip
        pip install pytest pytest-golden ruff mypy pyyaml

    - name: Lint with ruff
      run: |
        ruff check . --config pyproject.toml
        ruff format --check .

    - name: Type check with mypy
      run: |
        mypy . --ignore-missing-imports

    - name: Run tests
      run: |
        pytest integration_test.py -v --tb=short
```
